# MoXing KV Cache 量化性能实测报告

## 测试环境

- **设备**: MacBook Pro M4 (12GB 统一内存)
- **模型**: carstenuhlig/omnicoder-9b:latest (Q4_K_M, 支持 262K 上下文)
- **架构**: Qwen3.5 混合架构 (KV Cache + Recurrent Memory)
- **后端**: llama.cpp Metal (Flash Attention 启用)

## 测试方法

对比不同上下文大小下 F16 vs Q4_0 (TurboQuant) 的性能：
- 内存占用 (RSS)
- 推理速度 (tokens/s)
- 首字延迟 (TTFT)

## 测试结果

### 32K 上下文

| 指标 | F16 | Q4_0 | 差异 |
|------|------|------|------|
| 内存 (MB) | 6,790 | 6,098 | **-692 MB (-10.2%)** |
| 速度 (tok/s) | 6.8 | 7.1 | +0.3 (+4.4%) |
| TTFT (s) | 0.95 | 0.90 | -0.05 |

**结论**: 小上下文时节省约 10% 内存，速度略有提升。

---

### 64K 上下文

| 指标 | F16 | Q4_0 | 差异 |
|------|------|------|------|
| 内存 (MB) | 7,788 | 6,387 | **-1,401 MB (-18.0%)** |
| 速度 (tok/s) | 7.4 | 7.4 | +0.0 (+0.3%) |
| TTFT (s) | 1.16 | 0.89 | **-0.27s** |

**结论**: 中等上下文时节省 18% 内存，TTFT 改善明显。

---

### 128K 上下文

| 指标 | F16 | Q4_0 | 差异 |
|------|------|------|------|
| 内存 (MB) | 9,134 | 6,962 | **-2,172 MB (-23.8%)** |
| 速度 (tok/s) | ❌ 失败 | 13.5 | - |
| TTFT (s) | - | 0.95 | - |

**结论**: F16 推理因内存压力失败，Q4_0 正常工作。**KV Cache 量化使大上下文推理成为可能**。

---

### 256K 上下文 (模型最大支持)

| 指标 | F16 | Q4_0 | 差异 |
|------|------|------|------|
| 内存 (MB) | 5,501* | 8,124 | - |
| 速度 (tok/s) | ❌ 失败 | 7.3 | - |
| TTFT (s) | - | 0.96 | - |

*F16 进程异常退出，内存值不准确。

**结论**: 仅 Q4_0 能在 256K 上下文下正常推理。

---

## KV Cache 内存分析

根据 llama.cpp 日志：

| 上下文 | KV Cache 类型 | KV Buffer Size | 节省 |
|--------|---------------|----------------|------|
| 64K | F16 | 2,048 MB | - |
| 64K | Q4_0 | 576 MB | **1,472 MB (72%)** |
| 128K | F16 | 4,096 MB* | - |
| 128K | Q4_0 | 1,152 MB | **2,944 MB (72%)** |

*理论值，实际测试时 F16 推理失败。

**关键发现**: KV Cache 量化单独节省 **72%** 的 KV 内存！

---

## 推荐配置

根据测试结果，推荐使用以下配置：

### 日常使用 (≤64K 上下文)
```bash
moxing ollama serve carstenuhlig/omnicoder-9b:latest \
  --kv-cache q4_0 -c 65536
```
- 内存节省 18%
- 速度保持不变
- TTFT 改善 23%

### 长文档处理 (128K-256K 上下文)
```bash
moxing ollama serve carstenuhlig/omnicoder-9b:latest \
  --kv-cache q4_0 -c 262144
```
- **必需**: F16 无法运行
- 内存节省 72% (KV Cache 部分)
- 速度稳定 7-13 tok/s

---

## 结论

1. **KV Cache 量化有效节省内存**：72% 的 KV 内存节省
2. **速度影响极小**：实测差异 <1%
3. **TTFT 可能改善**：Q4_0 更快的内存访问
4. **大上下文必需**：128K+ 上下文时 F16 无法运行，Q4_0 正常工作

**建议**：在所有场景下默认使用 `--kv-cache q4_0`。
